# Full-Stack Web App — Image Analytics Dashboard

Extend the existing serverless pipeline (S3 → Lambda → Rekognition → DynamoDB) into a **full-stack web application** with a browser-based dashboard where users can upload images, view analysis results, and browse their image library — all within AWS Free Tier ($0).

## User Review Required

> [!IMPORTANT]
> **Frontend hosting approach:** This plan uses **S3 static website hosting** (free) instead of CloudFront. This means the dashboard URL will be an S3 website endpoint like `http://my-cs-ai-dashboard.s3-website-us-east-1.amazonaws.com`. CloudFront would add HTTPS + a cleaner URL but adds complexity. Is S3-only hosting acceptable?

> [!IMPORTANT]
> **Upload method:** Users upload images through the browser via **S3 presigned URLs** generated by the API. This avoids routing large files through API Gateway (which has a 10MB payload limit) and goes directly to S3. The existing S3 → Lambda → Rekognition pipeline then kicks in automatically.

> [!WARNING]
> **CORS configuration required:** The S3 source bucket and API Gateway both need CORS headers configured so the browser frontend can interact with them. This will be handled in Terraform.

## Open Questions

> **Q1: Do you want the frontend built with a framework (React/Vue) or plain HTML/CSS/JS?**
> - **Plain HTML/CSS/JS** — simpler, no build step, single file deploy, easier to understand
> - **React (via Vite)** — component-based, more portfolio-impressive, requires `npm run build` before deploy
>
> This plan assumes **plain HTML/CSS/JS** for simplicity. Let me know if you'd prefer React.

> [!IMPORTANT]
> **Q2: S3 bucket naming** — S3 bucket names are globally unique. The plan uses `my-cs-ai-dashboard` for the frontend bucket. You may need to change this if it's already taken.

## Proposed Changes

### Component 1: API Lambda (Backend)

New Lambda function that serves as the API backend, queried by the frontend through API Gateway.

#### [NEW] [api_function.py](file:///c:/Source%20Code/Automated%20Image%20Analytics/api_function.py)

A new Lambda handler with three route handlers:

| Route | Method | Action |
|---|---|---|
| `/images` | `GET` | Scans DynamoDB, returns all analysis records (sorted by timestamp) |
| `/images/{id}` | `GET` | Gets a single record by `ImageID` |
| `/upload-url` | `POST` | Generates an S3 presigned PUT URL (valid 5 min) so the browser can upload directly |

- Uses `boto3` DynamoDB `scan()` and `get_item()`
- Uses `s3_client.generate_presigned_url()` for direct browser-to-S3 uploads
- Returns JSON with proper CORS headers (`Access-Control-Allow-Origin: *`)
- Error handling with structured JSON error responses

Static HTML/CSS/JS site hosted on S3, providing a visual interface for the analytics pipeline.

| `aws_api_gateway_stage` | `dev` stage |
| `aws_lambda_function.api` | API backend Lambda |
| `aws_lambda_permission.api_gateway` | Allow API Gateway to invoke API Lambda |
| `aws_iam_role.api_lambda_exec` | IAM role for API Lambda |
| `aws_iam_policy` × 3 | DynamoDB Scan/GetItem, S3 GetObject/PutObject (presigned URLs), CloudWatch Logs |

**Modifications to existing resources:**

- Add `s3:PutObject` permission to the source bucket for presigned upload URLs (scoped to API Lambda role only, NOT the processor Lambda)
- Add CORS configuration to the source images S3 bucket
- Add new outputs: `frontend_url`, `api_gateway_url`

**IAM — Least Privilege for API Lambda:**

| Permission | Scope |
|---|---|
| `dynamodb:Scan`, `dynamodb:GetItem` | `ImageAnalysisMetadata` table ARN only |
| `s3:PutObject` | `my-cs-ai-source-images/*` (for presigned upload URLs) |
| `s3:GetObject` | `my-cs-ai-source-images/*` (for presigned download URLs) |
| `logs:*` | API Lambda log group only |
